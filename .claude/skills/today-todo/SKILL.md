# 오늘 할 일 정리 (today-todo)

`work-files/schedule/`의 일정 메모와 `work-files/meetings/`의 회의록을 함께 읽고, 오늘 챙겨야 할 일을 정리하는 스킬입니다.

## 동작 순서

1. 오늘 날짜(`YYYY-MM-DD`)를 확인합니다.
2. `work-files/schedule/` 폴더에서 오늘 날짜 파일(`YYYY-MM-DD.txt`)을 찾아 읽습니다.
   - 폴더나 오늘 날짜 파일이 없으면 오늘 등록된 일정이 없다는 사실을 그대로 알립니다.
3. `work-files/schedule/` 폴더 안의 다른 날짜 파일들도 확인해서, 오늘 이후 대략 1주일 이내에 마감이나 일정이 있는 항목을 찾습니다. (곧 다가오는 일 후보)
4. `work-files/meetings/` 폴더 안의 모든 회의록 파일(`.txt`)을 읽습니다.
   - 폴더나 파일이 없으면 회의록이 없다는 사실을 그대로 알립니다.
   - 회의록에서 담당자와 할 일이 함께 명시된 액션 아이템만 추출합니다 ("~하기로", "~가 챙기기로", "~가 준비하기로" 등).
   - "아직 못 정함", "협의 중", "확정 아님", `??`로 표시된 미정·불확실한 항목은 할 일 목록에서 제외합니다.
5. 각 파일의 `*급한 거*` / `*잊지 말기*` 표시 항목은 우선적으로 챙겨서 반영합니다.
6. 아래 세 가지로 구분해서 정리합니다.
   - **오늘 마감인 일**: 오늘 날짜가 마감·기한으로 명시된 항목 (일정 파일, 회의록 모두 포함)
   - **곧 다가오는 일**: 오늘 이후 며칠 내로 다가오는 일정·마감
   - **회의에서 나온 할 일**: 회의록에서 나온 담당자별 액션 아이템 (마감일이 있으면 함께 표시, 없으면 생략)
   - 해당 카테고리에 내용이 없으면 "없음"이라고 명시합니다.

## 출력 규칙

- 항상 한국어로, 공손하고 간결하게 작성합니다 (CLAUDE.md의 답변 말투 규칙을 따름).
- 위 세 가지 소제목으로 구분해서 목록으로 제시합니다.
- 항목마다 담당자(있는 경우)와 출처 파일(날짜/회의명)을 간단히 표시합니다.
- 원문에 없는 내용은 추측하거나 지어내지 않습니다.
- 미정·불확실한 사항은 할 일 목록에 넣지 않되, 필요하면 별도로 "확인 필요" 정도로만 짧게 덧붙일 수 있습니다.
