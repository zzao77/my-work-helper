# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 참고하는 안내 문서입니다.

## 저장소 목적

이 저장소는 소프트웨어 프로젝트가 아닙니다 — 코드, 빌드, 린트, 테스트 도구가 존재하지 않습니다. 이 저장소는 `work-files/` 아래에 있는 개인 업무 문서 작업 공간으로, Claude Code를 업무 비서로 활용해 일상적인 사무 문서(일정, 회의록, 보고서, 거래처 연락처)를 한글로 작성하고 정리하는 데 사용됩니다.

## 구조

- `work-files/schedule/` — 하루 단위 할 일/일정 메모, 파일명은 `YYYY-MM-DD.txt`
- `work-files/meetings/` — 회의록, 파일명은 `YYYY-MM-DD_회의주제.txt`
- `work-files/reports/` — 마크다운으로 작성된 장문 보고서 (예: `h1-work-report.md`)
- `work-files/contacts/` — 거래처별로 하나씩, 회사명으로 파일명을 짓고 회사명·담당자·연락처 등을 담은 파일

## 사용자 정보 및 답변 규칙

- 역할: 총무팀 사무 담당
- 자주 하는 일: 일정 정리, 회의록 요약, 보고서 초안 작성
- 답변 말투: 항상 한국어로, 공손하고 간결하게. 추측하지 말고 모르면 모른다고 답할 것.

## 콘텐츠 작성 규칙

- 모든 내용은 가상의 샘플/연습용 데이터이며, 실제 회사나 개인 정보가 아닙니다. 모든 파일은 `(교육용 가상 샘플 — 실명·실제 정보 아님)` 문구로 시작하며, 새로 만들거나 수정하는 모든 파일에서 이 문구를 유지해야 합니다.
- 파일은 한글로 작성하며, 격식 있는 비즈니스 문체가 아니라 편하게 손으로 쓴 메모 느낌(구어체 어미, 말줄임표, 괄호 속 잡담, 불확실함/급함을 나타내는 `??`/`!!`)으로 씁니다 — 일정/회의록 파일은 이 톤을 맞춰야 합니다.
- 보고서(`.md` 파일)는 좀 더 체계적으로, 맨 위에 결론(요약) 섹션을 먼저 쓰고 그 뒤에 번호가 매겨진 `##` 섹션, 마지막에 남은 할 일을 적는 `TODO:` 줄로 구성됩니다.
- 일정/회의록 파일(`.txt`)은 느슨한 구조의 메모입니다: 시간순 항목 목록 뒤에 "급함/잊지 말기" 표시 섹션(`*급한 거*` / `*잊지 말기*`)이 따라옵니다.
- 파일 간 서로 참조할 때(예: 회의록에 언급된 거래처, 보고서에 언급된 마감일) 참조되는 파일들 사이에서 이름/날짜를 일관되게 유지해야 합니다.
- 파일명뿐 아니라 본문 내용에 등장하는 날짜도 항상 `YYYY-MM-DD` 형식으로 씁니다 (예: `2026-07-06`).
- 코드마다 상세한 주석 추가
